# 🛠️ C Project Template

Универсальный шаблон для быстрого старта C-проектов. Включает в себя настроенный `Makefile`, интеграцию с библиотекой юнит-тестирования `check`, генерацию отчётов покрытия кода (`gcov`/`lcov`) и готовую конфигурацию для VS Code.

## ✨ Возможности

- 🏗️ Сборка статической библиотеки.
- 🧪 Запуск юнит-тестов с использованием библиотеки [`check`](https://libcheck.github.io/check/).
- 📊 Генерация HTML-отчёта покрытия кода (Code Coverage).
- 💅 Автоматическое форматирование кода через `clang-format`.
- ⚙️ Готовая конфигурация IntelliSense для VS Code (macOS / Apple Silicon).

## 📋 Требования

Для работы вам понадобятся:
- Компилятор `gcc` или `clang`
- `make`
- Библиотека `check`
- `lcov` (для генерации отчётов покрытия)
- `clang-format` (для форматирования)

**Установка зависимостей (macOS / Homebrew):**
```bash
brew install check lcov clang-format
```

**Установка зависимостей (Ubuntu / Debian):**
```bash
sudo apt-get install check lcov clang-format
```

## 📂 Структура проекта

```text
.
├── README.md               # Описание проекта
├── misc/                   # Дополнительные материалы (данные, картинки и т.д.)
└── src/                    # Исходный код проекта
    ├── Makefile            # Скрипт сборки
    ├── main.c              # Точка входа для основного приложения
    ├── NAME.c              # Исходный код вашей библиотеки
    ├── NAME.h              # Заголовочный файл
    └── test/
        └── test.c          # Файл с юнит-тестами
```

> ⚠️ **Важно:** Перед использованием не забудьте заменить `NAME` и `TEST` в `Makefile` на реальные названия ваших файлов!

## 🚀 Использование

> ⚠️ Все команды сборки и тестирования выполняются из директории `src/`.

Перейдите в папку с исходниками:
```bash
cd src
```

### Доступные команды Makefile

| Команда | Описание |
| :--- | :--- |
| `make` или `make all` | Собирает статическую библиотеку (`NAME.a`). |
| `make main` | Собирает исполняемый файл `app` из `main.c` и библиотеки. |
| `make test` | Компилирует и запускает юнит-тесты из `test/test.c`. |
| `make gcov_report` | Запускает тесты с флагами покрытия, генерирует HTML-отчёт в папку `report/` и открывает его в браузере. |
| `make format` | Форматирует все `.c` и `.h` файлы в проекте с помощью `clang-format`. |
| `make clean` | Удаляет объектные файлы (`*.o`), библиотеки и исполняемые файлы. |
| `make fclean` | Выполняет `clean` + удаляет все файлы генерации покрытия (`*.gcda`, `*.gcno`). |

## 💻 Настройка VS Code

В репозиторий уже добавлена папка `.vscode` с файлом `c_cpp_properties.json`. 
Он настроен для **macOS (Apple Silicon)** и включает пути к заголовочным файлам библиотеки `check` (`/opt/homebrew/opt/check/include`), чтобы IntelliSense корректно подсвечивал код и не выдавал ошибок при импорте `check.h`.

Если вы используете другую ОС или архитектуру, просто обновите `compilerPath` и `includePath` в этом файле под себя.
